# A股日K 5日收益预测建模

适用场景：

- 基于日K历史数据做 `未来5个交易日收益率` 预测
- 给 `每只股票` 生成预测值，不做组合、不做 TopN 组合回测
- 生成两份标准输出文件：
  - `每只股票预测结果.csv`
  - `每天预测准确率统计.csv`
- 后续生成的数据默认放在 skill 自带的 `outputs/` 目录内
- 需要做滚动锚点回测，验证模型在历史不同日期上的预测表现

## 核心原则

- 训练和验证都只能使用锚点日当时可见的数据。
- 不做离线拟合。预测某个 `锚点日期` 时，训练样本必须同时满足：
  - `日期 < 锚点日期`
  - `卖出日 <= 锚点日期`
- 也就是说，只有已经走完未来5个交易日、真实涨跌结果在锚点日已经可知的历史样本，才能进入训练集。
- 默认剔除 `今天尚未完成` 的最新交易日，避免把未收盘K线混入样本。
- 预测结果分两项：
  - `预测涨跌`
  - `预测5日收益率`
- 准确性也分两项：
  - `涨跌预测是否准确`：方向对记 `1`，否则 `0`
  - `涨跌幅度预测是否准确`：`|预测5日收益率 - 实际5日收益率| <= 5%` 记 `1`，否则 `0`
- 方向准确率默认提供两种口径：
  - 全量方向准确率：所有有5日后标签的股票都参与统计。
  - 降噪后方向准确率：`|实际5日涨跌幅| <= 2%` 的样本视为方向噪声，不参与分母；这是当前优化目标的主看口径。
- 每日准确率统计必须基于 `当日全部股票`，不是组合，也不是 Top3。

## 标准运行方式

1. 滚动锚点回测，输出两份标准文件：

```bash
bash skills/a-share-kline-return-modeling/scripts/run_model.sh
```

滚动到指定锚点截止日：

```bash
bash skills/a-share-kline-return-modeling/scripts/run_model.sh --anchor-end-date 2026-05-06
```

2. 指定某个锚点日，只输出该日全部股票预测：

```bash
python3 scripts/train_5d_return_model.py --as-of-date 2026-05-13
```

3. 只跑5日后涨跌方向模型：

```bash
python3 scripts/train_5d_direction_model.py --as-of-date 2026-04-24
```

方向模型默认使用核心关键特征：

```bash
# 核心关键特征
python3 scripts/train_5d_direction_model.py --as-of-date 2026-04-24 --feature-set core

# 如需临时验证旧口径，可显式使用样本表里的全部数值字段
python3 scripts/train_5d_direction_model.py --as-of-date 2026-04-24 --feature-set all
```

`core` 模式只保留价格、量能、短中期动量、均线位置、波动、K线形态、20日区间位置、市场/行业环境、核心指数相对强弱等关键字段，用于降低噪声和过拟合风险。

4. 训练和回测5日后市场方向模型：

```bash
python3 scripts/train_5d_market_direction_model.py --test-dates 2026-03-16,2026-03-19,2026-03-24
```

市场方向模型每天一行，预测股票池未来5个交易日整体偏涨还是偏跌。默认从 `00_A股日K基础行情与5日后标签表.csv` 和 `00_5日方向模型特征表.csv` 按 `日期 + 代码` 合并生成市场样本。训练口径同样禁止未来泄漏：预测某个锚点日期时，训练样本必须满足 `日期 < 锚点日期` 且 `卖出日 <= 锚点日期`。回测评估时才允许使用未来5日真实上涨比例判定是否预测成功。

市场模型支持两种决策模式：

```bash
# 旧版二分类融合
python3 scripts/train_5d_market_direction_model.py --test-dates 2026-04-27,2026-04-28 --decision-mode baseline

# 上涨比例回归 + 五档市场状态 + 多窗口
python3 scripts/train_5d_market_direction_model.py --test-dates 2026-04-27,2026-04-28 --decision-mode regime
```

`regime` 模式会输出 `预测未来5日上涨比例`、`机会模型强上涨概率`、`风险模型强下跌概率`、`融合预测上涨比例`、`预测市场状态`。当前该模式仍处于实验阶段，不能直接用于约束个股模型。

市场样本特征包括：

- 市场宽度：当日上涨/下跌比例、大涨/大跌比例、接近涨停/跌停比例
- 横截面分布：涨跌幅均值、中位数、标准差、分位数、偏度、极差
- 成交量能：成交额/成交量均值、中位数、标准差及滚动变化
- 行业扩散：上涨行业比例、强涨/强跌行业比例、行业分化度
- 指数背离：沪深300、中证500、中证1000相对强弱，小盘/中盘/大盘背离
- 连续性和位置：连续强弱、连续涨跌、近5/10/20日市场宽度和涨跌幅位置
- 风险状态：恐慌释放、过热回落风险、弱势延续风险

当前结论：市场方向模型不直接作为个股预测的裁决器。市场模块优先用于：

- 给每个锚点日打市场环境标签和风险标签
- 分市场环境统计个股5日方向模型准确率
- 识别个股模型容易失效的市场环境

按市场环境统计个股模型表现：

```bash
python3 scripts/analyze_stock_accuracy_by_market_env.py
```

输出文件：

- `09_个股预测结果_附市场环境标签_YYYYMMDD_YYYYMMDD.csv`
- `09_按实际市场状态统计个股预测准确率_YYYYMMDD_YYYYMMDD.csv`
- `09_按市场风险标签统计个股预测准确率_YYYYMMDD_YYYYMMDD.csv`
- `09_按日期与市场环境统计个股预测准确率_YYYYMMDD_YYYYMMDD.csv`

基于市场环境做个股方向风险修正：

```bash
python3 scripts/apply_market_risk_correction.py --correction-policy v15
```

该脚本不是用市场模型直接替代个股模型，而是在个股模型先给出 `预测涨跌` 后，只用锚点日当时可见的市场宽度、指数位置和风险标签做二次风险修正。当前投用的修正场景包括：

- 当前投用版本：`v15`。这是跨年稳定版高置信修正层，核心目标是减少方向修正规则对单月极端市场的误杀。
- `恐慌释放高+弱势延续高且大中小盘20日趋势转弱`：`v15` 不再直接把原始上涨强制改为下跌。2025-10-17 回测显示该规则会在恐慌后强反弹日造成大幅误杀，因此保留原始模型判断。
- `过热回落高且指数高位延伸`：核心指数处在20日高位或20日涨幅过大，且个股上涨概率低于阈值时，把原始上涨改为下跌。
- `常规环境但短线破位`：市场上涨比例低、平均涨跌幅为负，上证指数10日趋势为负且20日位置接近低位时，把原始上涨改为下跌。
- `常规环境但弱趋势强反弹`：中小盘20日趋势仍弱，但当日市场出现强反弹时，把原始上涨改为下跌。
- `弱势延续且当日继续大面积下跌`：弱势延续风险高、当日上涨比例低且中小盘20日趋势为负时，把原始上涨改为下跌。
- `常规环境但高位上涨乏力`：核心指数处在20日高位，但当日上涨比例、平均涨幅和行业强涨比例不够强时，把原始上涨改为下跌。
- `强反弹但上涨置信度不足`：当日市场强反弹且存在过热回落风险，但个股上涨概率低于50%、上证20日趋势不弱且中证1000未明显强趋势时，把原始上涨改为下跌。
- `弱势反弹但个股弱于行业`：弱势延续风险高、当日是大面积反弹，但个股相对行业偏弱且上涨概率不强时，把原始上涨改为下跌。
- `趋势回调但行业相对强`：市场短线回调但20日趋势仍为正，且个股明显强于行业时，把部分原始下跌改为上涨。`v15` 额外要求当日上涨比例在 `25%~45%`、当日平均涨跌幅大于 `-0.5%`、上涨行业比例大于 `45%`，避免强下跌日过度乐观改上涨。
- `高位强趋势且行业强`：指数处在高位强趋势，行业上涨比例和行业平均涨幅较强时，把部分原始下跌改为上涨。
- `上升趋势中恐慌洗盘`：只有恐慌释放高、没有弱势延续高，且核心指数20日趋势仍为正时，把部分原始下跌改为上涨。
- `恐慌但弱势延续未形成`：当日恐慌释放但弱势延续特征不足，且中小盘20日位置没有完全破位时，把部分原始下跌改为上涨。

注意：`实际市场方向`、`实际市场状态`、`市场未来5日上涨比例` 只用于回测判卷和结果解释，不能作为预测修正条件。

输出文件：

- `10_个股预测结果_市场风险修正_YYYYMMDD_YYYYMMDD.csv`
- `10_市场风险修正总体对比_YYYYMMDD_YYYYMMDD.csv`
- `10_按市场风险标签对比修正效果_YYYYMMDD_YYYYMMDD.csv`
- `10_按日期对比市场风险修正效果_YYYYMMDD_YYYYMMDD.csv`

### v16 信号出手层

`v15` 修正层之后，默认使用 `v16` 信号层做低覆盖、高置信出手筛选：

```bash
python3 scripts/apply_signal_decision_layer.py \
  --detail-csv skills/a-share-kline-return-modeling/outputs/10_个股预测结果_市场风险修正_训练清洗_2025core_v15_10月全月_20251009_20251031.csv \
  --output-prefix skills/a-share-kline-return-modeling/outputs/14_训练清洗_2025core_v15_v16信号_10月全月 \
  --decision-policy v16
```

`v16` 不追求全股票覆盖，而是只在回测中更稳定的高置信环境出手。当前投用规则只做上涨信号：

- `v16_过热强势延续上涨`：过热回落高环境下，原始方向仍为上涨、预测置信度足够，且当日市场宽度强、中证1000近20日趋势仍强。
- `v16_弱势修复上涨`：弱势延续高环境下，原始方向为上涨、预测置信度足够，且当日上涨比例没有过低。
- `v16_恐慌弱势反抽上涨`：恐慌释放高叠加弱势延续高环境下，原始方向为上涨且预测置信度足够。

当前目标口径从“全股票方向准确率70%～80%”调整为“低覆盖率出手信号准确率70%～80%”。全股票方向预测继续作为打分底座，不作为交易信号直接使用。

`00_5日涨跌方向预测样本明细.csv` 是 5日涨跌方向预测的业务主表。每一行表示“某只股票在某个交易日收盘后，预测未来5个交易日后是上涨还是下跌”的一个训练/验证样本。当前主表已删除冗余特征，只保留 61 列，其中 49 列可进入模型训练。它包含：

- 原始日K字段：开盘、收盘、最高、最低、成交量、成交额、振幅、涨跌幅、换手率等
- 5日后标签：卖出日、5日后收盘、5日后的实际涨跌幅、5日后实际涨跌
- 价格趋势特征：1/3/5/10/20日收益率、5/10/20日均线乖离
- 量价特征：5/10日成交量比、成交额比、1日成交量/成交额变化
- 风险和形态特征：5/10日波动率、振幅均值、实体、上下影线、收盘位置、20日区间位置
- 市场/行业特征：全市场上涨比例、全市场平均涨跌幅、行业平均涨跌幅、行业上涨比例、个股强于行业
- 指数相对强弱：上证指数、沪深300、中证500、中证1000以及个股相对核心指数强弱

字段定义见 `references/标准输出字段说明.md`。

## 数据分层

为了避免一个宽表同时承载事实、标签和模型输入，后续优先按两张表管理：

- `00_A股日K基础行情与5日后标签表.csv`：基础事实表，保留 `日期 + 代码`、股票名称/行业/板块/交易所、A股日K字段，以及5日后卖出日、5日后收盘、5日后的实际涨跌幅、5日后实际涨跌等判卷标签。
- `00_5日方向模型特征表.csv`：模型特征表，保留 `日期 + 代码` 和当前5日方向模型使用的核心特征字段。

两张表用 `日期 + 代码` 作为关联主键。预测建模时从特征表取模型输入，从基础事实表取股票信息和回测判卷标签。

拆分脚本：

```bash
python3 scripts/split_5d_model_tables.py
```

## 修改或排查时怎么做

1. 先确认日K文件列名是否仍为：`日期、开盘、收盘、最高、最低、成交量、成交额、振幅、涨跌幅、涨跌额、换手率`
2. 再确认股票元数据至少包含：`代码、名称、行业`
3. 若用户强调“不能用今天的数据”，不要手工把今天K线加回去
4. 若用户强调“不要离线拟合”，必须检查训练集过滤条件包含 `卖出日 <= 锚点日期`
5. 若用户要的只是结果文件，不额外输出旧版 `Top3`、组合回测、上涨概率文件
